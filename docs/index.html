RepoSage Documentation

Version: 1.3.0 Node: 18+ License: MIT GitHub

Minimal reference for installation, commands, options, generated output, and practical CLI usage examples.

Install

npm install -g reposage
# or
pnpm add -g reposage
reposage --version
reposage --help

CLI Commands

Command Purpose Example
reposage scan <path> Fast repository scan with counts and key files reposage scan .
reposage analyze <path> Structured analysis of stack, tooling, architecture signals reposage analyze ./my-project
reposage generate <path> Generate Cursor artifacts and docs from analysis reposage generate . --no-ai
reposage explain <path> Human-readable project summary reposage explain .
reposage update Update global installation or print source update guidance reposage update

Generate Options

Option Description Example
--ai Use Cursor CLI for AI-backed generation reposage generate . --ai
--no-ai Use built-in Handlebars templates only reposage generate . --no-ai
-m, --model <model> Pass model name to Cursor CLI (default gpt-5.2) reposage generate . --model gpt-5.2
-f, --force Overwrite existing generated files without prompt reposage generate . --force

Quick Usage

# 1) Scan
reposage scan .

# 2) Analyze
reposage analyze .

# 3) Generate artifacts
reposage generate . --no-ai

# 4) Explain repository summary
reposage explain .

AI Usage

# Install Cursor CLI
curl https://cursor.com/install -fsS | bash

# Authenticate
agent login
agent status

# Generate with AI
reposage generate .
reposage generate . --model gpt-5.2

If Cursor CLI is unavailable, use --no-ai.

Generated Output

  prompts/

  automations/

docs/
  context/
  • Rules: architecture, testing, security, naming, tech-stack constraints.
  • Commands: explain, trace, document, generate tests, analyze performance.
  • Prompts: feature generation, refactoring, debugging, review, unit tests.
  • Context docs: architecture overview, module index, glossary, known risks.

More Examples

# Analyze a different repo path
reposage analyze ../another-repo

# Force refresh generated output after refactor
reposage generate . --force --no-ai

# AI generation with explicit model
reposage generate ./service-a --model gpt-5.2

# Short summary for onboarding
reposage explain ./service-a

Notes

  • Architecture and module inference are evidence-driven and heuristic.
  • README.md is not part of the normal generate pipeline.
  • Use generated output as a starting point, then refine with team conventions.
