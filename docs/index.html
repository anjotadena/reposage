Daily Development Workflow

How to use the generated Cursor setup for maximum productivity

1

Starting Your Day

Quick context refresh

  • • Open .cursor/context/project-overview.md to remember project structure
  • • Review .cursor/context/architecture-overview.md for component relationships
  • • Check high-signal files listed in project overview

Useful commands

# Run the explain-repo command in Cursor
# Or use the trace-feature command to understand a flow
2

Implementing Features

Use prompts for quality

Reference generate-feature.md prompt when building new functionality. It structures AI output for implementation plans, code, and tests.

Follow the rules

Cursor automatically applies .cursor/rules/*.mdc to constrain AI behavior. Architecture, security, and testing rules keep output consistent.

Create modules

Use create-module and create-endpoint commands to scaffold new code following existing patterns.

3

Code Review & Testing

Automation workflows

Maintenance & Releases

Usage Guide

Complete reference for RepoSage CLI and generated artifacts

CLI Commands

reposage generate <path>

Generate all Cursor artifacts for a repository

--force Overwrite existing files
--no-ai Use templates instead of AI
--model <m> Specify AI model
reposage scan <path>

Scan repository and report file statistics

reposage analyze <path>

Analyze repository structure and tech stack

reposage explain <path>

Output human-readable repository summary

Best Practices

  • ✓ Run on new repos: Generate artifacts when starting on an unfamiliar codebase
  • ✓ Commit generated files: Share the .cursor/ directory with your team
  • ✓ Refresh periodically: Re-run generation after major refactors or architecture changes
  • ✓ Customize rules: Edit generated rules to add project-specific constraints
  • ✓ Use automations: Reference automation workflows for recurring tasks like PR review
  • ✓ Check context first: Read project-overview.md before diving into code

Artifact Reference

Detailed breakdown of each generated artifact type

R

Rules (.cursor/rules/*.mdc)

Durable constraints that guide AI behavior

▼

Rules are always-on context that Cursor applies to every interaction. They constrain what the AI does and ensure consistency.

Core rules

  • • 00-repo-baseline — Project fundamentals
  • • 10-architecture — Module boundaries, patterns
  • • 20-testing-strategy — Test frameworks, coverage
  • • 30-security — Security findings, constraints
  • • 40-tech-stack — Languages, frameworks, tools

Extended rules

  • • coding-standards — Style, formatting, quality
  • • api-design — REST conventions, patterns
  • • naming-conventions — Files, functions, types
C

Commands (.cursor/commands/*.md)

Manual workflows you invoke on-demand

▼

Commands are step-by-step workflows you trigger manually. Use them for specific development tasks.

Understanding code

  • • explain-repo — Get a project overview
  • • trace-feature — Follow a feature through layers
  • • document-module — Generate module docs

Building & reviewing

  • • create-module — Scaffold new modules
  • • create-endpoint — Add API endpoints
  • • generate-tests — Write tests for code
  • • review-risk — Security/architecture review
  • • analyze-performance — Find bottlenecks
A

Automations (.cursor/automations/*.md)

Recurring workflows for common tasks

▼

Automations are structured workflows for recurring tasks. They include triggers, steps, expected outputs, and fallback behaviors.

Review & Security

  • • pr-review-or-risk-scan — Comprehensive PR review
  • • security-review — Security-focused analysis
  • • dependency-change-review — Audit dependency updates

Maintenance

  • • test-plan-on-large-change — Generate test plans
  • • refresh-context-on-structure-change — Update context docs
  • • release-readiness-check — Pre-release validation
